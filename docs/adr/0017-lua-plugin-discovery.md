# hex/lua/plugin is the escape hatch ADR-0007 reserved

[ADR-0007](0007-lua-runtime-only.md) kept `hex/lua` a bare runtime and pushed plugin discovery to consumers, with an explicit note that if one needed it, we'd "extract that on demand into `hex/lua/plugin` or a separate module — not up front." This is that extraction, scoped to exactly one shape: repo-local Cobra commands written in Lua, Teal, or Fennel, discovered from a `.hex/command/` directory and merged into an existing command tree.

`hex/lua` core is unchanged. `hex/lua/plugin` is additive: a manifest format (`config.toml`), an entrypoint convention (`run.{lua,tl,fnl}` + optional `args.{lua,tl,fnl}`), directory-nesting-as-command-nesting, and a small set of Go→Lua bindings (`disk`, `argv`/`argc`, `cmd`, `dump`, `explode`, `sleep`) that make a bare script useful as a command. None of this lives in `hex/lua` itself — `hex/lua/plugin` is just another consumer of `Environment`, same as any app-specific binding would be.

The shape directly replicates a sibling project's proven Lua-plugin mechanism rather than designing something bespoke, with two deliberate deltas: discovery is repo-local (`.hex/command/`, relative to cwd) instead of a global `~/.config/<app>/` directory, since hex plugins are meant to travel with the repo they extend; and entrypoints support Teal and Fennel in addition to Lua for free, because `hex/lua`'s `Environment.LoadFile`/`ExecFile` already auto-detects and compiles all three by extension — the plugin loader never branches on language.

This first pass covers commands only. `.hex/generator/` (scripted `hex make` generators, using the same mechanism with a different mount convention) is a deliberate, deferred follow-up — not a gap in this design. `RuntimeOption`/`WithModule` (`lua/plugin/runtime.go`) exist so a consumer app with a running `*hex.App` can preload its own bindings (e.g. `config`, `env`) into every plugin invocation without `hex/lua/plugin` needing to know about `hex.App` at all.

No sandboxing changes accompany this: the per-invocation `Environment` opens the full Lua stdlib, same posture that sibling project ships with. A plugin script has the same trust level as code checked into the repo it lives in — this is not a boundary for untrusted third-party code, and should not be treated as one without further hardening (`hex/lua`'s `WithoutStandardLibraries` is available to a consumer that wants to lock it down).
